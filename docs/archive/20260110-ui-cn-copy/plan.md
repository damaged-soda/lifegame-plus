# Web UI 全量中文文案 技术变更计划（Plan）

## 0. 背景与目标（简短即可）
- 背景：当前页面存在中英混用的 UI 文案（按钮、标题、提示、状态等），影响一致性与可读性。
- 目标：将 Web UI 所有用户可见文案统一为中文（保留必要的符号/缩写，例如 `B.../S...`、`n=3`、倍率 `x10`）。
- 非目标：
  - 不做 i18n 多语言切换（仅中文一套）
  - 不改交互逻辑与布局，仅替换文案（除非为了中文长度做极小样式适配）

## 1. 影响范围（必须）
- 影响的 repo（来自 docmap，可多项）：lifegame-plus
- 影响的模块/目录/文件（按 repo 分组列出即可）：
  - `src/lifegame-plus/web/index.html`：静态标题/分区标题/提示文案
  - `src/lifegame-plus/web/main.js`：运行时文案（按钮文本、状态提示、下拉选项文本等）
  - （如需要）`src/lifegame-plus/web/style.css`：为中文长度做少量样式调整（可选）
- 外部可见变化：
  - 页面所有 UI 文案改为中文（标题、按钮、面板标题、提示、状态）

## 2. 方案与改动点（必须）
说明：实现将按批次推进；每批次写入前需先列出本批次改动文件/影响并征求确认。

- repo: lifegame-plus
  - 改动点：
    - 统一页面文案为中文，预计包含但不限于：
      - 顶部/页面标题：`Rules` 等
      - 按钮：`Play`/`Pause`、`Reset`
      - 速率选择：`Speed` 与各倍率选项（倍率符号可保留）
      - 规则面板状态：`Paused`、`Playing (pause to edit)`
      - 规则面板分区：`Rule Sets`、`Atomic Rules`、`Birth`、`Survive`、`Current`
      - 规则提示：邻居摆放示意提示（Note...）
      - 规则集下拉：
        - `Custom`
        - 经典规则集名称（可保留常见英文专名，但整体 UI 文案应为中文；或提供“中文名（英文名）”）
  - 新增/修改的接口或数据结构：无
  - 关键逻辑说明：仅替换字符串常量与 DOM 文案，不改变行为。

## 3. 自测与验收口径（必须，可执行）
- 本地自测步骤：
  1. 根目录：`python3 -m http.server 8000`
  2. 打开：`http://127.0.0.1:8000/src/lifegame-plus/web/`
  3. 浏览页面各区域，确认无英文残留（允许保留 `B.../S...`、`n=...`、倍率 `x...`）
  4. 点击 `播放/暂停`、`重置`、切换 `速率`、切换 `规则集`/`原子规则`，确认文案更新不影响功能
- 关键用例清单：
  - 所有按钮/标签/提示/状态均为中文
  - 规则字符串与技术符号按需保留（`B.../S...`、`n=`、`x`）
  - 原有功能不回归
- 通过标准：
  - 页面没有遗留英文 UI 文案（除约定保留项）
  - 控制台无新报错
