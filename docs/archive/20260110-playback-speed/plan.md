# 播放速率下拉组件（Web）技术变更计划（Plan）

## 0. 背景与目标（简短即可）
- 背景：当前 Web MVP 仅支持 `Play/Pause`，无法调节演进速度，体验与调试不便。
- 目标：在 `Play/Pause` 之外新增“播放速率”下拉组件，提供：x0.5 / x1 / x2 / x5 / x10 / x20 / x50 / x100；切换后需立即生效（播放中也即时生效）。
- 非目标：不做速度滑条、不中断渲染优化、不做状态编辑/重置/随机等其他功能；不引入第三方依赖与打包链路。

## 1. 影响范围（必须）
- 影响的 repo（来自 docmap，可多项）：lifegame-plus
- 影响的模块/目录/文件（按 repo 分组列出即可）：
  - `src/lifegame-plus/web/index.html`：新增速率 `<select>`
  - `src/lifegame-plus/web/main.js`：播放循环支持速率倍率，并支持播放中即时切换
  - `src/lifegame-plus/web/style.css`：布局微调（按钮 + 下拉在同一行或同一块区域）
- 外部可见变化：页面出现速率下拉；播放中可随时切换速率

## 2. 方案与改动点（必须）
说明：实现将按批次推进；每批次写入前需先列出本批次改动文件/影响并征求确认。

- repo: lifegame-plus
  - 改动点：
    - UI：新增速率下拉，默认 x1
    - 逻辑：播放循环按 `speedFactor` 调整演进速率；切换下拉后立即生效
  - 新增/修改的接口或数据结构：无（仅修改前端播放循环与 UI）
  - 关键逻辑说明（实现口径）：
    - 采用“渲染与演进解耦”的循环：固定渲染节奏（如 `requestAnimationFrame`），按时间累积执行 `step` 若干次以匹配目标 steps/s；从而支持 x100 并避免极小 `setInterval` 带来的不稳定。
    - `x1` 以当前默认速度为基准，其它倍率按比例缩放。

## 3. 自测与验收口径（必须，可执行）
- 本地自测步骤：
  1. 根目录：`python3 -m http.server 8000`
  2. 打开：`http://127.0.0.1:8000/src/lifegame-plus/web/`
  3. 点击 `Play` 开始演进；在播放中切换速率：x0.5 → x1 → x10 → x100，观察速度立即变化；再次点击暂停应停止演进
- 关键用例清单：
  - 播放中切换速率立即生效（无需重新点击 Play）
  - 暂停时切换速率不触发演进；再次 Play 以新速率运行
  - 下拉包含且仅包含 8 个选项：0.5/1/2/5/10/20/50/100
- 通过标准：
  - UI 出现速率下拉且默认选中 x1
  - 速率切换即时生效，`Play/Pause` 行为不回归
